# template.pptx（卒論）書式マップ

同梱 `assets/template.pptx` = 2025/02/05 卒論発表PPTX。体裁の本体はここのマスター/レイアウトにある。

## 基本仕様
- スライドサイズ：**9906000 × 6858000 EMU（10.83 × 7.5 in、4:3寄りの独自サイズ）**。変更しない。
- アクセント色：**濃紺 `000F78`**（見出しの縦バー・Indexシェブロン・表紙の区切り線）。
- 既定フォント：日本語ゴシック（出力時は Yu Gothic 相当）。本文は黒。
- マスターは1つ。レイアウトは5種（下記）。

## レイアウト（`prs.slide_layouts`）
| 名前 | 用途 | プレースホルダ |
|---|---|---|
| **`1行タイトル+本文`**（index 0） | **本文スライドの基本**。見出し＋本文＋ページ番号 | idx0=TITLE(見出し, 左上・濃紺縦バー付き), idx1=BODY(本文, ■/●/→の箇条書き), idx12=SLIDE_NUMBER |
| `1_1行タイトル+本文` | 同上の別バリエーション | 同上 |
| `BLANK` | 表紙・Index（図形を直接配置） | なし |
| `TITLE_ONLY` | 見出しのみ | TITLE |
| `中表紙` | 章区切り（セクション扉） | TITLE |

→ 本文は **`1行タイトル+本文` でスライドを足し、TITLE と BODY のプレースホルダを埋める**だけで
濃紺見出し・箇条書き体裁を継承できる。`slidekit.Deck.content()` がこれを行う。

## 元テンプレのスライド構成（13枚）
| # | 種別 | 備考 |
|---|---|---|
| 0 | 表紙(BLANK) | タイトル(36pt・2行)＋濃紺区切り線＋研究室名(20pt)＋学籍/氏名(20pt)＋日付 |
| 1 | Index(BLANK) | 「00.Index」見出し＋濃紺シェブロンの GROUP × 6 |
| 2–10,12 | 本文(`1行タイトル+本文`) | 見出し「0X.章名 X-Y.小見出し」＋本文＋図/表 |
| 11 | 中表紙 | 章扉 |

`slidekit` は **表紙(0)とIndex(1)を再利用**（文字だけ差し替え／Indexはシェブロン作り直し）、
本文は新規にレイアウトから足し、最後に**元の本文スライド(2..)を削除**する。

## 箇条書きの階層（BODY プレースホルダ）
- level 0 → ■（第1階層）
- level 1 → ●（第2階層）
- level 2 → →（補足）
`content()` の bullets で `(level, "文")` のタプルにすると階層が付く。記号はレイアウト由来なので
自分で「■」を文字として付けない。

## 設計書 → スライドの対応づけ
スライド設計書（例 `中間発表_スライド設計書_v6.docx`）は「スライド1枚＝1ブロック」で
【キーメッセージ／ボディ／チャート】が書いてある。対応：
- **キーメッセージ** → 見出し直下の最初の箇条書き（結論を1文で）。曖昧表現（核心/本丸）は使わない。
- **ボディ** → ■/● の箇条書き（短く）。
- **チャート** → `images=[...]`。実物PNGがある図は貼る、未作成図は `placeholder=` で枠。
  - 例：dBc/Hz=`05_benchmark_data/bench_p2_curated.png`、結果=`worklog/.../260610/model_kpe0.08_PN.png`・`model_HOLD_PN.png`、
    制御図=`fig4_block.png`/`fig_ctrl.png`、感応度=`fig3_plant.png`、1/N=`fig_pw_1overN.png`、配置=`fig_pw_matrix.png` 等。

## レイアウト別の図配置（`content(layout=...)`）
- `"right"`：本文を左半分に狭め、図1枚を右に（横長の図でも可）。
- `"below2"`：本文を上に短く、図2枚を下に左右並べ（FBループ＋特性、結果2条件 など）。
- `"below"`：図1枚を下に大きく。
- `placeholder="…"`：図の代わりに灰色の「作成予定」枠。

## 既知の注意（python-pptx）
- スライド削除は「新規追加の**後**」に行う（先に削除→後で追加すると slideN.xml 重複で破損）。`slidekit.save()` が順序を担保。
- 日本語フォントは run の `a:rPr` に `a:latin` と `a:ea` を両方手動設定（`get_or_add_ea` は無い）。`slidekit._ea` 参照。
- 右に図を置くときは BODY プレースホルダの幅を狭める（重なり防止）。`content(layout="right")` が自動でやる。
