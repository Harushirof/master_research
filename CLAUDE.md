# CLAUDE.md - master_research プロジェクトガイド

## 研究テーマと概要

水晶振動子（クォーツ発振器）の**周波数同期制御**に関する修士研究。

2台の発振器の周波数差（FRFR: Frequency Ratio / Phase差 [ns]）をリアルタイム計測し、
NI DAQ からの制御電圧によって一方の発振周波数を調整することで位相同期（周波数ロック）を実現する。

### 主な実験内容

- **周波数差（FRFR）のリアルタイム計測**: SIGLENT SDS2204X オシロスコープから VISA (LAN) で FRFR [ns] を取得
- **フィードバック制御**: P 制御 → PID 制御 → 状態空間フィードバック（2状態: FRFR + dFRFR/dt）へ段階的に発展
- **パルス応答実験**: 電圧パルス（振幅・パルス幅スイープ）に対する FRFR の変化量 (ΔFRFR) を計測し、系のゲイン K [ns/(V*s)] を同定
- **位相アンラップ**: FRFR が 0-100 ns の周期で折り返すため、ジャンプ検出によるアンラップ処理を実装
- **状態空間制御**: 車両追従制御のアナロジーに基づく 2 状態フィードバック（位相誤差 + 周波数誤差）

### ハードウェア構成

| 機器 | 役割 |
|------|------|
| NI DAQ (Dev1) | アナログ出力 (ao0, ao1) で制御電圧を印加 |
| SIGLENT SDS2204X (192.168.1.61) | FRFR 計測（MEAS:ADV:P3:VAL?） |
| 水晶振動子 x2 | 制御対象（周波数源） |

### 実験系

```
OCXO-A (10MHz, 基準) ──→ オシロ CH1 ──┐
                                        ├─ FRFR計測 ──→ PC ──→ FBロジック
OCXO-B (10MHz, 制御対象) → オシロ CH2 ─┘                        │
       ↑                                                         │
       └──── NI USB-6211 (ao0/ao1) ←────────────────────────────┘
```

- FRFR = CH1→CH2 立ち上がりエッジ間の時間差 [ns]。0-100ns で周期ラップ（10MHz = 周期100ns）
- 同期成功 = FRFR がオシロ上で停止

### 研究フェーズ

| Phase | 目標 | 状態 |
|-------|------|------|
| 1 | FRFR を任意の一定値で停止 | 達成済み（251203 v3: 10回連続成功、std < 0.4ns） |
| 2 | FRFR を**指定した値**に停止 | 現在取り組み中 |
| 3 | 2つの OCXO それぞれに電圧を与えて調整 | 未着手 |
| 4 | ミキサーを使った同期 | 未着手 |

## 使用ツール

- **Git / GitHub** (`Harushirof/master_research`)

## データ形式

- **実験ログ**: CSV (`writetable` で出力)
  - 典型的な列: `time_s`, `frfr_corrected_ns`, `ao1_V`, `drift_ns`, `freq_err_ns_per_s`, `dv_V`
- **解析結果**: CSV (サマリテーブル)、PDF/FIG (プロット)
- **バイナリ**: `.mat` (複数Run結果の一括保存)
- **注意**: CSV/PDF/PNG 等のデータファイルは `.gitignore` で除外。Git には `.m` スクリプトのみ追跡する

## コーディング規約

### 図の保存
- `exportgraphics` を使う（`saveas` は非推奨）
- 形式は PDF または PNG
```matlab
exportgraphics(fig, 'results/figures/plot_name.pdf', 'ContentType', 'vector');
exportgraphics(fig, 'results/figures/plot_name.png', 'Resolution', 300);
```

### スクリプト構成
- 実験スクリプトは **function** として記述（ワークスペース汚染を防ぐ）
- DAQ / VISA の後始末は `onCleanup` + `cleanupDAQ` 関数で確実に行う
- 電圧出力は必ず `clamp(v, v_min, v_max)` で制限する
- レートリミット (`du_max`) を適用し、急激な電圧変化を防ぐ

### 命名規則
- スクリプト: `frfr_<実験種別>_<日付>_v<版>.m`
- ログCSV: `frfr_<種別>_<タイムスタンプ>.csv`
- worklog: `worklog/<学期>/<YYMMDD>/` に日付ごとに格納

### 変数・単位
- FRFR: `[ns]` (ナノ秒)
- 制御電圧: `[V]`（0-5V）
- 周波数誤差: `[ns/s]`
- ゲイン K: `[ns/(V*s)]`

## 勤怠管理（チェックイン / チェックアウト）

研究室の稼働は時間制（最低160h）。`worklog/timesheet.md` で記録を管理する。

### 運用ルール

ユーザーが **「チェックイン YYYY-MM-DD HH:MM」** と言ったら:
1. `worklog/timesheet.md` のテーブルに新しい行を追加（チェックアウト欄は空欄）
2. 「チェックインしました（YYYY-MM-DD HH:MM）」と返答

ユーザーが **「チェックアウト YYYY-MM-DD HH:MM」** と言ったら:
1. `worklog/timesheet.md` の最新の未完了行のチェックアウト欄と稼働時間を埋める
2. 累計稼働時間を計算して報告
3. 「チェックアウトしました。本日 X.Xh / 累計 XX.Xh（目標160h）」と返答

日付・時刻はユーザーが指定する。省略や曖昧な指定があれば確認する。

### timesheet のフォーマット

```markdown
| 日付 | チェックイン | チェックアウト | 稼働時間 | メモ |
|------|-------------|---------------|---------|------|
| 2026-04-21 | 14:00 | 17:30 | 3.5h | FB制御実験 |
```

## Claude Code への注意事項

- **`.m` ファイルの実行はできない**: Claude Code は MATLAB エンジンを持たないため、スクリプトの生成・編集・レビューのみ対応。実行はユーザーが MATLAB 上で行う
- **DAQ / VISA 接続コードの動作確認不可**: ハードウェア依存のコードは構文チェックのみ可能。実機テストはユーザーが行う
- **worklog 内のファイルは履歴的な記録**: 直接編集せず、新しいバージョンを作成する。整理済みコードは `src/` に配置する
- **CSV データの読み込みは可能**: `Read` ツールでデータ内容を確認し、解析スクリプトの作成を支援できる
- **Simulink (.slx) ファイルはバイナリ**: 内容を読むことはできない
- **日本語コメント推奨**: ユーザーの母語に合わせて日本語でコメントを記述する
