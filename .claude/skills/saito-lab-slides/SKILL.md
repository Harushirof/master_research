# 齋藤研スライド（卒論フォーマット）作成スキル

## このスキルの要点（なぜテンプレ継承か）

卒論PPTXの体裁（濃紺アクセントの見出し・フォント・■/●/→ の箇条書き・濃紺シェブロンのIndex・
表紙の区切り線）は、**同梱の `assets/template.pptx` のスライドマスター／レイアウトに入っている**。
python-pptx で図形を手作りして再現しようとすると、色・位置・フォントが必ず微妙にズレて
「それっぽいけど違う」ものになる（一度それで失敗している）。

だから本スキルは **テンプレートを開き、そのレイアウトでスライドを足してプレースホルダを埋める**。
これで本物の体裁をそのまま受け継ぐ。ヘルパー `scripts/slidekit.py` がこの流れを実装している。

## 使い方（基本フロー）

`scripts/slidekit.py` の `Deck` を使う。図は事前に PNG にしておき（後述）、内容は設計書から取る。

```python
import sys; sys.path.insert(0, ".../.claude/skills/saito-lab-slides/scripts")
from slidekit import Deck

d = Deck()                                  # 同梱テンプレートを開く（表紙・Index は元スライドを再利用）
d.set_title(["水晶振動発振器の位相同期制御", "による高純度信号作成"],
            student="福井 晴士郎", date="2026/06/24")     # 表紙の文字を差し替え（指定したものだけ）
d.set_index(["01 研究背景", "02 本研究の目標", "03 先行研究", "04 研究方法",
             "05 実験と結果", "06 考察と今後の展望", "Appendix"])   # Index のシェブロンを作り直す

# 本文スライド：見出し / 箇条書き / 図。bullets は文字列(第1階層=■) か (level, 文字列)。
d.content("01 研究背景：基準信号の純度と現状",
          ["位相雑音＝位相のゆらぎ。純度が高いほど性能が向上",
           (1, "原子時計は最高純度だが大型・高価"),
           (1, "OCXOは短期安定に優れるが単体に限界")],
          images=["/abs/path/bench.png"], caps=["市販各機の位相雑音"], layout="right")

d.content("04-5 制御ロジック", ["…", "…"],
          images=["/abs/loop.png", "/abs/char.png"], caps=["FBループ", "入出力特性"], layout="below2")

d.content("04-2 実験系", ["制御系統と評価系統の二系統"],
          placeholder="実験系ブロック図\n（清書予定）")    # 図が未作成のときは灰色プレースホルダ

d.save("中間発表_スライド_v2.pptx")          # 仕上げ：元テンプレの本文スライドを削除して保存
```

- `layout`: `"right"`（右に1枚）／`"below2"`（下に2枚）／`"below"`（下に1枚大きく）。
- `placeholder=...` を渡すと図の代わりに「○○：作成予定」の灰色枠を置く（未作成図の明示に使う）。
- 中表紙（章区切り）が要るときは `d.divider("04 研究方法")`。

詳しい書式仕様・スライド型・設計書からの対応づけは **`references/template-map.md`** を読むこと。

## 図の用意

スライドに貼る図は **事前に PNG にしておく**（`slidekit` は図を作らない。貼るだけ）。
- 既存の解析図（例：`worklog/.../260610/model_kpe0.08_PN.png`、`05_benchmark_data/bench_p2_curated.png`）はそのまま使う。
- 説明図は matplotlib で作る。日本語は **Yu Gothic**（`font_manager.addfont` で `C:\Windows\Fonts\YuGothM.ttc` 等を登録）。本研究の図生成例は `presentations/mid_term_202606/04_slides_draft` 周辺の `gen_*assets.py` を参照。
- まだ無い図は `placeholder=` で枠だけ置き、残タスクとして残す。

## 内容は設計書から取る

スライドの内容（各スライドのキーメッセージ・箇条書き・どの図か）は、リポジトリの
**スライド設計書（例：`中間発表_スライド設計書_v6.docx`）**に「スライド1枚＝1ブロック」で書いてある。
設計書のキーメッセージは見出し下の最初の箇条書き（結論）として置くとよい。曖昧表現（「核心」「本丸」等）は使わない。

## QA（必須）

体裁スキルなので**必ず目視確認する**。`anthropic-skills:pptx` スキルの手順に従い:
1. `soffice --headless --convert-to pdf` で PDF 化 → `pdftoppm`/PyMuPDF で各スライドを PNG に。
2. **サブエージェント（新しい目）で全スライドを点検**：本文のはみ出し、図と箇条書きの重なり、
   キャプションとページ番号の衝突、余白、図の歪み。
3. 指摘された user-visible な不具合（はみ出し・重なり・欠落）だけ直す。1サイクルで止める。

## よくある落とし穴

- **本文プレースホルダに図を右置きするときは `body.width` を狭める**（`content(layout="right")` は自動でやる）。
  しないと本文が図に重なる。
- **新スライドは末尾に足し、元の本文スライドの削除は `save()` で最後に行う**（先に削除して後から足すと
  python-pptx が slideN.xml を重複させて壊れる）。`slidekit` はこの順序を守っている。
- **日本語フォントは latin/ea の両方に設定**（python-pptx に `get_or_add_ea` が無いので手動。`slidekit._ea` 参照）。
- 1スライトに詰め込みすぎない。箇条書きは短く、結論は1文。
