# 状態方程式による制御の可能性 — 整理と今後の展望

**作成**: 2026-04-21
**位置づけ**: Phase 2 に向けて、これまでの状態空間FB試行（251216, 260109）を振り返り、改めて設計方針を整理する。

## 1. なぜ状態空間制御を検討するか

### 1.1 背景の比喩（車両追従問題）

2台のOCXOを同期する問題は、車両追従制御のアナロジーで考えられる:

| 車両追従 | OCXO 同期 |
|---------|-----------|
| 車間距離 `d` | 位相差 `FRFR` |
| 速度差 `Δv` | 周波数差 `dFRFR/dt` |
| アクセル開度 | ao1 電圧 |
| 目標: 一定車間かつ同じ速度 | 目標: 指定位相差かつ周波数一致 |

251216 では `blog_reproduction/acc_simulation.m` で車両追従の状態空間制御をシミュレートし、2状態フィードバック `u = -K·x + r` で安定収束する様子を確認している。

### 1.2 2状態で考える自然さ

- **位相誤差 `e_phase = FRFR - FRFR_ref`** を 0 に
- **周波数誤差 `e_freq = dFRFR/dt`** を 0 に
- 2つを独立に抑え込めば完全同期（位置も速度も合う）

これは状態方程式で書くと:

```
x = [FRFR; dFRFR/dt]
ẋ = A·x + B·u,  y = C·x
A = [0 1; 0 0],  B = [0; K]  (積分器＋直結)
```

制御則 `u = -K·x`（状態フィードバック）で極配置ができるので、整定時間と減衰比を自由に設計できる。

## 2. これまでの試行と失敗の分類

| 試行 | フォルダ | 構造 | 結果 |
|------|---------|------|------|
| v1 | 251216/equation_of_state_1216_v1 | 絶対値指令 比例 | 定常偏差残る |
| v2 | 251216/equation_of_state_1216_v2 | 絶対値指令 比例 | 78 ns に収束（目標25ns）→ **ずれる** |
| v3 | 251216/equation_of_state_1216_v3 | v2 + wrap 処理 | wrap が誤差の単調性を破壊 → **発散** |
| v4 | 251216/equation_of_state_1216_v4 | Δf→0 (freq_err のみ制御) | 値制御なし。Phase 1 の再表現 |
| 260109 | 260109/frfr_state_space_fb_2state_run.m | 2状態FB 絶対値指令 | 飽和 → 制御権喪失 → **発散** |

### 2.1 共通の失敗原因 — 「絶対値指令」の落とし穴

上記すべての失敗に共通する構造:

```matlab
u_cmd = u0 - k1*e_phase - k2*e_freq     % ← これが絶対値指令
```

問題点:
- 真に同期する平衡電圧 `u_eq` は未知（OCXO固体差・環境温度で変わる）
- `u0 = 1.54 V` は単なる仮の基準値。`u_eq` との差 `u0 - u_eq` は補正できない
- 結果: 定常偏差 `e_ss = (u0 - u_eq) / k1` が必ず残る → 目標値からずれたところで平衡する
- もし `k1` を大きくすると、過渡で `u` が飽和 → 制御権を失って発散

**要点: 状態FBであっても、「絶対値指令」には積分作用がなく、未知平衡点を補償できない。**

### 2.2 Phase 1 が成功したのに Phase 2 で失敗した理由

Phase 1 (`run_frfr_fb_single.m`) の制御則 `v_ao1 = v_ao1 + dv` は**積分型更新**。
Phase 2 初期試行 `u_cmd = u0 - k1·e - k2·ė` は**絶対値指令**。

この構造の違いが本質。Phase 1 は暗黙的に積分器が入っている（積分プラントに対して積分制御 → 型1系になる）ので平衡電圧を学習できた。

## 3. 現行 Phase 2 設計（260421 v1）

### 3.1 制御則

```
e_phase  = FRFR_ref - FRFR_unwrapped   [ns]
freq_err = dFRFR/dt                    [ns/s]
delta_u  = Ki · e_phase - Kd · freq_err [V]
u[k]     = clamp(u[k-1] + delta_u, 0, 5)  ← 積分型更新
```

### 3.2 状態空間的な解釈

拡張状態を `x_aug = [FRFR; dFRFR/dt; u]` と取ると、`u[k]=u[k-1]+Δu` は `u` を状態として扱うことに等しい。制御則は:

```
Δu[k] = [Ki, -Kd, 0] · [FRFR_ref-FRFR; -dFRFR/dt; 0]
      = -[Ki, -Kd] · [e_phase; e_freq] (符号反転込み)
```

つまり **「位相誤差＋周波数誤差の2状態 × 電圧積分器」** という3次系の極配置問題。
これは古典制御でいう PI-D 構造に相当する。

### 3.3 設計値

| パラメータ | 値 | 意味 |
|-----------|----|------|
| Ki | 0.0003 V/ns | 位相誤差を引っ張るゲイン |
| Kd | 0.0018 V/(ns/s) | 周波数ドリフト減衰ゲイン |
| du_max | 0.05 V/step | レートリミット |
| ω_n | 0.3 rad/s | 設計固有振動数 |
| ζ | 0.8 | 設計減衰比 |
| 整定時間 | ≈ 15 s | 2%整定 |
| 閉ループ極 | z = 0.924 ± 0.039j | 離散極（\|z\| = 0.925）|

## 4. より高度な状態空間制御へのロードマップ

Phase 2 が動いた後に検討する発展案:

### 4.1 LQR（最適レギュレータ）

`Q, R` を重み行列として `u = -K_lqr · x` を自動設計。
重みの意味:
- `Q(1,1)` 大 → 位相誤差を早く潰す
- `Q(2,2)` 大 → 周波数誤差を早く潰す
- `R` 大 → 電圧動作を控えめに

MATLAB `dlqr()` で一発設計できる。現行の手調整ゲインを置き換える候補。

### 4.2 状態オブザーバ（カルマンフィルタ）

現状 `dFRFR/dt` は単純差分＋指数平滑で推定（260109 の `alpha=0.2` 方式）。
- 計測ノイズが大きいと速度推定が荒れる
- カルマンフィルタで `[FRFR; dFRFR/dt]` を同時推定すれば、推定精度と遅延のトレードオフを理論最適にできる
- 先行研究（時刻スケール分野）で Kalman-based time scale algorithm が確立済み

### 4.3 積分器拡大（Servo Control）

状態に `∫e_phase dt` を加えて `u = -K·[e_phase; e_freq; ∫e_phase]` で制御。
これで外乱（温度ドリフト、経年変化）に対しても定常偏差 0 を保証できる。
現行 Phase 2 は実質これの連続時間版（Δu に `Ki·e_phase` 項がある）。

### 4.4 多発振器（Phase 3, 4）への展開

将来 N 台に拡張する場合の状態空間:

```
x = [FRFR_12; dFRFR_12/dt; FRFR_13; dFRFR_13/dt; ...; FRFR_1N; dFRFR_1N/dt]
u = [u_2; u_3; ...; u_N]   (N-1 個の制御入力)
```

- 次元: 2(N-1) 状態 × (N-1) 入力
- 基準 OCXO（1番）に対する他の OCXO の相対位相差をすべて制御
- MIMO 状態FBなので LQR が自然
- 発振器間に相互作用（Kuramoto 的なカップリング）はないので、各位相差が独立に扱える（疎行列）
