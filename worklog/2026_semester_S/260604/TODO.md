# 260604 作業メモ: FB 電圧変化の最小化（一気通貫・連続録音）

## 背景・診断
- 260521 録音より: FB 制御中は 0.6Hz のうねり ±0.3〜0.4 ns、HOLD（電圧固定）は ±0.15 ns。
- 原因: **電圧をいじること自体が位相をずらす**。狙い = ロックを保てる範囲で電圧変化を最小化。

## 方針
- #0,1,2（タイミング修正）は対象外。確認するのは **#3 ゆっくり / #4 平均 / 仕上げ（デッドバンド）**。
- 今は「どれが効くか絞る段階」 → 各設定 **1回ずつ**。反復は勝者が決まってから。
- HOLD は開ループ（制御と無関係）なので毎回不要 → セッション中 **2個だけ**（先頭と末尾）でドリフト確認。

## 実験: 一気通貫・連続録音（約25分）
録音は TASCAM で回しっぱなし。MATLAB は **1本実行するだけ**:
```matlab
frfr_phase2_sequence_260604_v1();      % 各区間5分。動作確認は o.slot_s=60
```
※ v0 は発散バグあり（先頭0Vマーカーが制御開始前に同期を壊した）。**v1 を使う**。
  v1 はマーカー廃止＋微分を 0526 と同じ /Ts に戻し、kijun 区間 = 0526 FB と完全一致。
区間順（各5分）:
1. kijun  FB（基準・今の設定で同期）
2. HOLD1  HOLD（基準①）
3. p3slow FB（Ts=1.0 ゆっくり）
4. p4avg  FB（nOS=5 平均）
5. gentle FB（Ts=1.0 + nOS=5 + deadband0.5 + du_max0.01）
6. HOLD2  HOLD（基準②）

- 先頭で 0V を10秒出す → 録音に 0V→u_init の段差が出る = WAV 上の「t=0」目印。
- **録音開始の時刻をメモ**（WAV と CSV の対応に使う）。
- 終了時に `..._segmap.csv`（各区間の t_start/t_end）と区間サマリ（std・電圧移動量）が出る。

## 評価
WAV を区間ごとに切り出して比較。窓 = segmap の t_start/t_end + （録音開始との時刻差）。
```matlab
% 例（offset = スクリプトt=0 と 録音開始 の差[s]）
m = eval_jitter_260604_v0("260604_xxxx.wav", t_start+offset+5, slot-10, 192000, true);
```
各 FB 区間の std・0.6Hz ピークを HOLD1/HOLD2 と比較。電圧移動量が小さく std が HOLD に近い設定が勝ち。
